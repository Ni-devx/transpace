**2.4.1.1. データ層**

1. **Storage API** — ノート・学習記録の読み書き権限を管理。Supabase RLSと連携し、ユーザーが承認した範囲のデータにのみアクセスを許可する。*例：フラッシュカード拡張機能がノートの本文を読み取り、問題を自動生成する。*
2. **Resource API** — URL・PDF・画像などの外部リソースをSupabaseに保存・取得する。*例：参考Webページをノートに紐付けて管理する。*
3. **Search API** — ノート・リソース・学習記録をまたいだ全文検索を提供する。*例：「微分」と検索すると、関連するノートと参照PDFが同時にヒットする。*

**2.4.1.2. UI層**

1. **UI API** — 拡張機能が描画できる領域をサイドバー・パネル・ポップアップに限定するサンドボックス設計。*例：ポモドーロ拡張機能がサイドバーにタイマーを描画する。*
2. **Editor API** — エディタのカーソル位置・選択テキスト・ブロック構造を取得・操作する。*例：AI添削拡張機能が選択中の文章を取得し、修正案をその場に挿入する。*

**2.4.1.3. イベント・状態層**

1. **Event API** — システムイベントをフックし、拡張機能側で処理を実行させる。*例：`note:saved`イベントをトリガーに、フラッシュカードを自動生成する。*
2. **Status API** — アプリのアクティブ状態・フォーカス状態を監視する。*例：アプリがバックグラウンドになった瞬間に学習セッションを自動終了する。*
3. **Telemetry API** — 学習時間・操作ログをユーザーに意識させずバックグラウンドで記録する。*例：ノートを開いていた時間を自動集計し、科目別の学習時間グラフを生成する。*

**2.4.1.4. スケジュール・通知層**

1. **Timer API** — 時間計測の共通基盤を提供する。*例：ポモドーロ拡張機能が25分タイマーを起動する。*
2. **Notification API** — プッシュ通知の送信権限をコア側で管理する。*例：SM-2スケジューラーが復習タイミングで「今日の復習カードが5枚あります」と通知する。*
3. **Calendar API** — 試験日・復習スケジュールをカレンダーに登録する。*例：試験日を登録すると、そこから逆算した復習スケジュールが自動でカレンダーに入る。*

**2.4.1.5. AI層**

1. **AI API** — TranspaceのLLMリソースに安全にアクセスし、独自プロンプトを実行させる。*例：英検対策拡張機能が「英検1級の語彙レベルで添削して」というプロンプトをAIに渡す。*
2. **OCR API** — 手書きテキスト・数式を高精度にテキスト化する。*例：手書きの数式をペンで囲むと、LaTeX形式に変換されてノートに挿入される。*

**2.4.1.6. 認証・権限層**

1. **Auth API** — ユーザーの認証状態・プロフィール情報へのアクセスを管理する。*例：拡張機能がユーザーIDを取得し、個人の学習データに紐付けて保存する。*
2. **Permission API** — 拡張機能がどのAPIにアクセスできるかをユーザーが制御する。*例：インストール時に「このパックはStorage APIとAI APIのみ使用します」と表示して許可を求める。*
3. **Extension API** ー 拡張機能全体関するAPI。*例：拡張機能公開プラットフォームの挙動を定義する。*
